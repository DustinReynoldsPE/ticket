# Format parser and serializer

Parse and serialize ticket markdown files with YAML frontmatter. This is the backward compatibility layer.

## Design

Files: pkg/ticket/format.go
Approach:
- Parse(reader io.Reader) (*Ticket, error): Split on --- delimiters, unmarshal YAML frontmatter via go-yaml, parse markdown body into sections by ## headings
- Serialize(ticket *Ticket) ([]byte, error): Emit canonical format - consistent field order, inline flow arrays, fixed section order (Description, Design, Acceptance Criteria, Notes)
- Read tolerance: go-yaml naturally handles both [a, b] and multi-line array forms
- Write normalization: always emit flow arrays, consistent field ordering
- Title from first # heading in body, or from frontmatter title field if present
- Notes parsing: each note is a timestamped entry under ## Notes section, format '- YYYY-MM-DDTHH:MM:SSZ: text'
- Handle edge cases: missing sections, empty frontmatter fields, tickets created by old bash version

## Acceptance Criteria

Round-trip: parse existing .tickets/ files and re-serialize without data loss. Normalized format on write.
