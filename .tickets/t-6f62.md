# Add 'tk move <repo>' to enable moving a ticket to a different project repo




Allow 'tk move <repo>' to move a ticket to a new repo. Check that directory includes a ticket repo before move. Mark original ticket as closed and add a note about move, new repo, time. Support --recursive --r to move parent and all children tickets, recursively. Default should be non-recursive? Need to decide before implementation what is the best approach.

## Design

Files: cmd/move.go, pkg/ticket/move.go

## Command
`tk move <id> <repo-path> [--recursive|-r]`

## Behavior
1. Validate target path has .tickets/ directory
2. Generate new ID using target repo's directory name prefix
3. Copy all content: description, design, acceptance, notes, tags, assignee, type, priority, status
4. Single move: set deps=[], links=[] on target ticket
5. Recursive (-r): collect full descendant tree (children, grandchildren, etc.)
   - Remap deps between co-moving tickets to new IDs
   - Remap parent references to new IDs
   - Strip deps/links pointing to non-moving tickets
6. Close original(s) in source with note: "Moved to <new-id> in <repo-path>. Stripped deps: [...]. Stripped links: [...]"
7. Add note on target: "Moved from <old-id> in <source-path>"

## ID Generation
Use GenerateID() scoped to target repo directory name for prefix derivation.

## Validation
- Target path must exist and contain .tickets/
- Ticket must exist in source
- For recursive: ticket should have children (warn if not?)
