# CLI commands: add-note, query, workflow, help

Implement remaining CLI commands: notes, JSON query, workflow guide, and help.

## Design

Files: cmd/notes.go, cmd/query.go, cmd/workflow.go
Approach:
- add-note id [text]: append timestamped note to ticket's Notes section. Support stdin pipe.
- query [jq-filter]: output all tickets as JSONL. If jq-filter provided, shell out to jq for filtering (jq remains optional dep). Without filter, native JSON output.
- workflow: print the workflow guide text (ticket types, statuses, readiness rules, status propagation). Static text, no computation.
- help: cobra generates this automatically from command annotations. Ensure output matches README usage section format.
- --json flag already handled at root level.

## Acceptance Criteria

add-note appends timestamped notes. query outputs valid JSONL. workflow prints guide. help matches README.
