# Rewrite tk in Go

Rewrite the tk CLI from a 2152-line bash script to Go for maintainability, performance, and distribution.

## Design

Architecture: Four layers sharing one core library.
- pkg/ticket: Core library (types, store, format, deps, filter, id, workflow)
- cmd/: CLI commands via cobra, exact same interface as bash version
- internal/tui/: Bubbletea TUI for interactive browse and edit
- internal/mcp/: MCP server for AI agent access via tk serve

File format: Keep markdown+YAML frontmatter in .tickets/. Tolerate old format on read, normalize on write (mutating ops only). go-yaml handles both array styles natively.

Distribution: GoReleaser for cross-compiled binaries (darwin/linux, arm64/amd64). Homebrew tap and AUR package updated to install binary instead of script.

Four phases: (1) Core + CLI parity, (2) TUI browse, (3) MCP server, (4) TUI edit.

## Acceptance Criteria

Phase 1: All 75 test-suite.sh assertions pass against Go binary
Phase 2: tk ui launches interactive ticket browser
Phase 3: tk serve starts MCP server, Claude Code can use it
Phase 4: TUI supports inline editing
