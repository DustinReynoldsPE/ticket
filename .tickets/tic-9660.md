# Add ls --group-by support


ls --group-by is missing from Go version. Bash supports: workflow, type, status, priority. Also --group as shorthand for --group-by=workflow.

## Design

Files: cmd/ls.go, pkg/ticket/filter.go
Add --group-by string flag and --group bool flag to ls command. Implement grouping logic:
- workflow: groups by in_progress / ready / blocked / needs_testing / closed (requires dep resolution)
- type: groups by epic, feature, task, bug, chore
- status: groups by status order
- priority: groups by P0-P4
Output: group headers between sections, tickets sorted within groups by priority then ID.

## Acceptance Criteria

tk ls --group-by=workflow matches bash output. All 4 group modes work. --group shorthand works. Add tests to test-suite.sh.
