# Status transitions and propagation

Implement status transition validation and parent status propagation when children change.

## Design

Files: pkg/ticket/workflow.go
Approach:
- ValidateTransition(from, to Status) error: enforce valid status transitions
- PropagateStatus(store Store, childID string) error: after child status change, check siblings:
  - All children closed → parent becomes closed
  - All children closed or needs_testing → parent becomes needs_testing
  - Otherwise parent stays in_progress
- Called from Update path when ticket has a parent
- Only propagates upward, never downward

## Acceptance Criteria

Status propagation matches test-suite.sh assertions for hierarchy gating and propagation.
