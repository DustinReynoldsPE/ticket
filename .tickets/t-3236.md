# FileStore CRUD and partial ID matching

Implement the filesystem-backed ticket store with create, get, update, delete, list, and partial ID resolution.

## Design

Files: pkg/ticket/store.go
Approach:
- FileStore struct with dir field (default .tickets, configurable via TICKETS_DIR)
- Create(t *Ticket): generate ID, serialize, write to dir/id.md, ensure dir exists
- Get(id string): resolve partial ID, read file, parse
- Update(t *Ticket): serialize to canonical format, write back (normalize on write)
- Delete(id string): resolve partial ID, remove file
- List(opts ListOptions): read all .md files in dir, parse each, return slice
- Resolve(partialID string): exact match first, then glob for *partialID*.md, error if ambiguous (multiple matches) or not found
- ListOptions struct: Status, Type, Priority, Assignee, Tag, Limit filters
- Performance: for List, read frontmatter only (stop at second --- delimiter) unless body needed
- Handle parent field: when creating with parent, add child ID to parent's children tracking

## Acceptance Criteria

CRUD operations work against real .tickets/ directory. Partial ID matching works for unambiguous prefixes.
