# TUI: inline editing capabilities

Add editing capabilities to the TUI: status changes, priority, assignee, notes, create, and deps.

## Design

Files: internal/tui/list.go (extend), internal/tui/detail.go (extend), internal/tui/form.go (new)
Approach:
- From list view:
  - s on selected ticket: cycle status (open → in_progress → needs_testing → closed)
  - p on selected ticket: popup priority selector (0-4)
  - c: switch to create form view
- From detail view:
  - s: cycle status
  - p: priority selector
  - a: set assignee (text input)
  - n: add note (text input)
  - d: dependency picker (shows ticket list, select to add/remove dep)
- form.go: new ticket creation form using bubbles/textinput
  - Fields: title, description, type (selector), priority (selector), assignee
  - Tab between fields, enter to submit, esc to cancel
- All mutations go through pkg/ticket Store, list refreshes after changes

## Acceptance Criteria

Can change status, priority, assignee from TUI. Can add notes and create tickets. Changes persist to disk.
